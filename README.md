# Деловые Решения — веб-обзор объектов и парка техники

Локальное приложение на **React + Vite + TypeScript**: главная с дашбордом, объекты, реестр спецтехники с карточками единиц, редактированием ТО, страховок, ремонтов и пропусков. Данные демо хранятся в браузере (**localStorage**), чтобы можно было сразу пользоваться без сервера.

## Быстрый старт

Требуется **Node.js 20+** (рекомендуется 22 LTS).

```bash
cd "Deloresh Objects"
npm ci
npm run dev
```

Откройте в браузере адрес, который покажет Vite (обычно **http://localhost:5173/**).

### С телефона в той же Wi‑Fi сети

Сервер уже слушает все интерфейсы (`0.0.0.0`). Узнайте IP компьютера (например, в macOS: «Системные настройки → Сеть») и на телефоне откройте:

`http://<IP-вашего-ПК>:5173/`

Либо используйте скрипт из репозитория (если настроен `cloudflared` и т.п.):

```bash
npm run dev:mobile
```

### Продакшен-сборка и локальный просмотр

```bash
npm run build
npm run preview
```

После `npm run preview` откройте в браузере **http://localhost:4173/** (порт **5173** занят режимом `npm run dev`).

Статика окажется в каталоге **`dist/`** — её можно раздавать любым статическим хостингом (Nginx, S3, Netlify, Cloudflare Pages и т.д.).

### Развёртывание на VPS (94.242.58.24) одной командой

Полный пошаговый гайд: **[docs/DEPLOY.ru.md](docs/DEPLOY.ru.md)**.
Подключение Telegram-группы как источника отчётов бригадира: **[docs/TELEGRAM-BRIDGE.ru.md](docs/TELEGRAM-BRIDGE.ru.md)**.

С вашего Mac, из корня проекта:

```bash
npm run deploy:server -- <ваш-логин>@94.242.58.24
```

Wrapper подключится по SSH, скачает свежий `server-bootstrap.sh` из
ветки `main` на GitHub и запустит его на сервере. Bootstrap идемпотентный:
ставит Node.js 20 + nginx, создаёт пользователя `deploy`, клонирует репозиторий, собирает фронт, ставит systemd-сервис `site-forms` (бэкенд заявок/отчётов/медиа) и nginx-конфиг с проксированием `/api → 127.0.0.1:8787`. После завершения шеф открывает `http://<IP>/`.

## Полезные команды

| Команда | Назначение |
|--------|------------|
| `npm run dev` | Разработка с HMR |
| `npm run build` | Проверка типов + production bundle |
| `npm run lint` | ESLint |
| `npm run test` | Vitest (один прогон) |
| `npm run test:watch` | Тесты в watch-режиме |
| `npm run check` | lint + test + build (как в CI) |
| `npm run launch` | Сборка + просмотр как на сервере (**http://localhost:4173/**) |

## Надёжность

- При ошибке рендера React показывается экран **«Сбой интерфейса»** с кнопкой обновления и ссылкой на главную (**Error Boundary** в `main.tsx`).
- Страницы подгружаются **лениво** (`React.lazy`) — быстрее первый экран, при переходе кратко показывается «Загрузка…».

## Данные и правки

- Моки и объединение с локальными правками — в коде (`src/data/`, хуки реестра техники).
- Правки в карточке техники сохраняются **локально в этом браузере**; сброс — кнопка «Сбросить» в режиме редактирования (где предусмотрено).

### Резервная копия localStorage (чтобы ничего не потерять)

После открытия приложения в консоли браузера (F12 → Console) доступен объект **`window.DELORESH_BACKUP`**:

- `download()` — скачать JSON с текущими данными приложения из storage;
- `exportJson()` — получить тот же текст строкой;
- `importJson(строка)` — восстановить из сохранённого файла;
- `keys()` — список задействованных ключей.

Подробно: **[docs/RECOVERY.ru.md](docs/RECOVERY.ru.md)** (восстановление кода, данных, что не в storage).

## CI

В GitHub Actions (файл `.github/workflows/ci.yml`) на push/PR в ветки `main` или `master` выполняются: `npm ci` → `lint` → `test` → `build`. Если основная ветка называется иначе — добавьте её в список `branches` в workflow.
