# Восстановление и резервные копии

## 1. Код проекта («всё слетело» в смысле файлов)

Исходники живут в папке проекта и (желательно) в **Git**.

- Если используете **Git** / GitHub: клонируйте репозиторий заново или откатите ветку (`git checkout`, `git reset` — только если понимаете последствия).
- Если Git **нет**: храните **zip-копию папки проекта** и периодически обновляйте её после важных изменений.
- Сборка **`npm run build`** создаёт только **`dist/`** — это не замена исходников, а готовый фронт для хостинга.

## 2. Ваши данные в браузере (localStorage)

Правки по технике, добавленные машины, пользовательские объекты и часть справочников сохраняются **только в том браузере**, где вы работали. Очистка сайта / кэша / «очистить данные» может **стереть** эти записи.

### Какие ключи использует приложение

| Ключ / префикс | Назначение |
|----------------|------------|
| `deloresh-user-sites:v1` | Объекты, созданные через «Добавить объект» |
| `fleet:registry` | Добавленная техника, удалённые из мока id, свои классы техники |
| `fleet:overrides:<id>` | Локальные правки по карточке единицы (ТО, страховка, ремонты, пропуска, паспорт) |
| `deloresh-procurement-requests:<siteId>:v1` | Заявки снабженцу по объекту (список заявок, чтобы не терялись после F5) |
| `deloresh-brigadier-reports:<siteId>:v1` | Отчёты бригадира по объекту (текст, проблемы, вложения в localStorage) |
| `deloresh-site-object-media-authors:<siteId>` | Запомненные ФИО авторов медиа по объекту |
| `deloresh-procurement-authors` | Запомненные ФИО в заявках снабжения |

При добавлении новых функций в коде список может расшириться — смотрите `src/lib/deloreshLocalStorageBackup.ts`.

### Резервная копия через консоль браузера

На любой странице приложения (после загрузки) в **инструментах разработчика → Console**:

```js
// Список ключей с данными
window.DELORESH_BACKUP?.keys()

// Скачать JSON-файл с резервной копией
window.DELORESH_BACKUP?.download()

// Или только текст в буфер: скопируйте вывод вручную
copy(window.DELORESH_BACKUP?.exportJson())
```

### Восстановление из файла

1. Откройте приложение на том же или новом компьютере (тот же origin, например `http://localhost:5173`).
2. В консоли вставьте **содержимое** сохранённого JSON (целиком файл) в переменную и импортируйте:

```js
const json = `… вставьте сюда содержимое файла …`
window.DELORESH_BACKUP?.importJson(json)
```

3. Перезагрузите страницу (`F5`).

Импорт записывает **только** разрешённые ключи; чужие поля в JSON игнорируются.

### Ручной способ (без моста)

В DevTools → **Application** → **Local Storage** → выберите origin → правый клик по ключам → Copy / Export (зависит от браузера). Можно сохранить значения в текстовый файл вручную.

## 3. Что не сохраняется в localStorage

- Исходные моки объектов и парка — они в **коде** (`src/data/`).
- Загруженные на сервер файлы (если сервера ещё нет) — в текущей архитектуре часть медиа может жить только в сессии; уточняйте по коду страницы объекта.

## 4. После сбоя интерфейса

Срабатывает **Error Boundary**: кнопка «Обновить страницу» и ссылка «На главную». Данные localStorage при этом **не очищаются** автоматически.
